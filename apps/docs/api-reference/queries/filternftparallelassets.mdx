---
title: "filterNftParallelAssets"
description: "Returns a list of Parallel assets based on a variety of filters."
---

**Returns:** [ParallelAssetFilterConnection](/api-reference/types#parallelassetfilterconnection)

### Arguments

<a href="/api-reference/input-objects#parallelassetfilters">
  <ResponseField name="filters" type="ParallelAssetFilters">
    A set of filters to apply.
  </ResponseField>
</a>

<ResponseField name="phrase" type="String">
  A phrase to search for. Can match a Parallel asset ID (`address`:`tokenId`) or partially match a name.
</ResponseField>

<a href="/api-reference/input-objects#parallelassetmatchers">
  <ResponseField name="match" type="ParallelAssetMatchers">
    A set of parameters to match.
  </ResponseField>
</a>

<a href="/api-reference/input-objects#parallelassetranking">
  <ResponseField name="rankings" type="[ParallelAssetRanking]">
    A list of ranking attributes to apply.
  </ResponseField>
</a>

<ResponseField name="limit" type="Int">
  The maximum number of Parallel assets to return.
</ResponseField>

<ResponseField name="offset" type="Int">
  Where in the list the server should start when returning items. Use `count`+`offset` from the previous query to request the next page of results.
</ResponseField>

### Schema


```graphql 
"""Returns a list of Parallel assets based on a variety of filters."""
filterNftParallelAssets(filters: ParallelAssetFilters, phrase: String, match: ParallelAssetMatchers, rankings: [ParallelAssetRanking], limit: Int, offset: Int): ParallelAssetFilterConnection
```
