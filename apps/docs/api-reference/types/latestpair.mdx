### Fields

<ResponseField name="address" type="String!" required>
  The contract address for the pair.
</ResponseField>

<ResponseField name="exchangeHash" type="String!" required>
  The contract address for the exchange.
</ResponseField>

<ResponseField name="id" type="String!" required>
  The ID of the pair (`address:networkId`).
</ResponseField>

<ResponseField name="initialPriceUsd" type="String!" required>
  The listing price, or first known price for the pair, in USD.
</ResponseField>

<ResponseField name="liquidAt" type="Int">
  The unix timestamp for when liquidity was added to the pair.
</ResponseField>

<ResponseField name="liquidity" type="String!" required>
  The total liquidity in the pair.
</ResponseField>

<ResponseField name="liquidityToken" type="String">
  The token with higher liquidity within the pair. Can be `token0` or `token1`.
</ResponseField>

<ResponseField name="networkId" type="Int!" required>
  The network ID the pair is deployed on.
</ResponseField>

<ResponseField name="newToken" type="String!" required>
  The newly added token within the pair. Can be `token0` or `token1`.
</ResponseField>

<ResponseField name="nonLiquidityToken" type="String">
  The token with lower liquidity within the pair. Can be `token0` or `token1`.
</ResponseField>

<ResponseField name="oldToken" type="String!" required>
  The pre-existing token within the pair. Can be `token0` or `token1`.
</ResponseField>

<ResponseField name="priceChange" type="Float!" required>
  The percent price change between the listing price and the current price.
</ResponseField>

<ResponseField name="priceUsd" type="String!" required>
  The newly added token price in USD.
</ResponseField>

  </ResponseField>

<ResponseField name="transactionHash" type="String!" required>
  The unique hash for the transaction that added liquidity, if applicable, otherwise the transaction that added the pair.
</ResponseField>
